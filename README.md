目前的运行逻辑:
  用户对话处理:user对话存入记忆库 --> 如果是图片调用qwen图像模型将其转换成文字描述，再存入记忆库中 --> 确保记忆库只有文本信息 --> 文本模型deepseek进行对话处理 --> 调用ai判断处理该对话 --> ai判断是否需要调用工具函数 --> 将调用tool函数的结果返回到记忆库中tool视角的记忆中，用于ai分析 --> 进行回复
  定时任务处理:程序启动时，timer_tool中的后台调度器主线程就在一直运行(只有一个) --> user对话请求定时任务 --> “用户对话处理” --> 发现需要调用timertool工具 --> ai判断定时任务需要调用的函数 --> 设置任务，创建线程池中的线程任务 --> 到达执行时间，进行唤醒(os计时器收到硬件中断，os处理中断，将sleepO中的线程池中的对应线程进行唤醒) --> 执行任务 --> 定时任务结束，自动销毁
